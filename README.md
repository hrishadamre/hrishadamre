## Project Highlights

<details open>
<summary><b>TriageIQ — AI-Powered Support Ticket Triage System</b></summary>

<br>

TriageIQ is an AI-assisted decision-support workflow for IT and customer support teams. The system analyzes incoming support tickets and helps agents decide what needs attention, where it should be routed, how risky it is from an SLA perspective, and how to respond.

**Core capabilities**

* Predicts whether a ticket is High, Needs Review, or Normal priority
* Recommends routing to the correct support group
* Scores SLA risk using transparent business rules
* Generates a first-response draft for human review
* Keeps the final decision with the agent instead of fully automating sensitive support actions

**Impact**

* Achieved approximately 95% high-priority recall
* Achieved approximately 81% grouped routing accuracy
* Designed the product around responsible AI and human-in-the-loop review
* Built a Streamlit prototype to demonstrate the full workflow

**Tools and concepts**
Python, machine learning, NLP, Streamlit, classification, routing logic, SLA risk scoring, AI product thinking, responsible AI

</details>

<details>
<summary><b>Airbnb Rating Prediction — End-to-End Machine Learning Pipeline</b></summary>

<br>

This project predicts whether an Airbnb listing will receive a perfect rating by using structured listing data, host behavior, pricing, availability, and text-based features.

**Core work**

* Processed 100K+ training listings and 12K+ test listings
* Engineered 50+ structured and text-based features
* Used TF-IDF and SVD for listing text representation
* Compared Logistic Regression, Decision Tree, kNN, Random Forest, and XGBoost
* Used nested holdout validation for more reliable model evaluation

**Impact**

* Achieved 78.13% accuracy and 0.828 AUC using XGBoost
* Identified rating drivers such as price, availability, host activity, and listing content
* Converted model results into recommendations for host and platform decision-making

**Tools and concepts**
Python, Pandas, NumPy, Scikit-learn, XGBoost, TF-IDF, SVD, feature engineering, classification, AUC/ROC

</details>

<details>
<summary><b>Terp Protect — Campus Crime Data Analysis & Relational Database Design</b></summary>

<br>

Terp Protect transforms campus crime data into a structured database and uses SQL analysis to identify safety patterns.

**Core work**

* Designed a normalized relational schema for incidents, locations, and crime categories
* Created primary and foreign key relationships to reduce redundancy
* Used joins, aggregations, filtering, and time-based queries
* Built visualizations to highlight high-risk areas and incident trends

**Impact**

* Helped identify frequently occurring incident types
* Supported analysis of time-based and location-based safety patterns
* Demonstrated how SQL and database design can support operational decision-making

**Tools and concepts**
SQL Server, relational database design, ER modeling, normalization, analytical querying, dashboarding

</details>

<details>
<summary><b>Financial Portfolio Optimization — Python Analytics</b></summary>

<br>

This project evaluates trading strategies and constructs an optimized investment portfolio using Python.

**Core work**

* Converted historical price data into return series
* Built Moving Average Crossover and Bollinger Band strategies
* Created an 8-strategy portfolio across 12 instruments
* Used correlation analysis, Monte Carlo simulation, and mean-variance optimization
* Evaluated return, volatility, Sharpe Ratio, drawdown, and beta

**Impact**

* Improved Sharpe Ratio from 1.02 to 1.22
* Reduced portfolio volatility to 8.38%
* Achieved low beta of approximately 0.118
* Demonstrated risk-aware decision-making instead of return-only evaluation

**Tools and concepts**
Python, Pandas, NumPy, Matplotlib, financial analytics, portfolio optimization, risk analysis

</details>

<details>
<summary><b>SSAA — Construction Workforce Scheduling Platform</b></summary>

<br>

SSAA is a client-focused system analysis project for a construction workforce scheduling platform.

**Core work**

* Gathered and structured client requirements
* Designed DFDs, ERDs, CRUD matrices, and role-based workflow models
* Modeled users, projects, assignments, subscriptions, availability, and notifications
* Helped refine MVP scope, onboarding strategy, and mobile-first workflows

**Impact**

* Improved clarity around manpower planning, project scheduling, and workforce availability
* Converted stakeholder needs into structured system requirements
* Demonstrated business analysis, product thinking, and scalable system design

**Tools and concepts**
Business analysis, requirements gathering, ERD, DFD, CRUD matrix, system design, workflow modeling, documentation

</details>

<details>
<summary><b>Drone-Based Human Drowning Detection</b></summary>

<br>

This project used drone-based image processing to support faster identification of swimmers in distress.

**Core work**

* Developed a Python/OpenCV and YOLOv8-based detection workflow
* Created and evaluated labeled data for distress vs. normal swimming behavior
* Integrated computer vision logic with the larger drone-based rescue concept
* Worked through hardware, software, and real-world testing challenges

**Impact**

* Achieved 93% detection accuracy
* Designed the solution to reduce manual surveillance effort
* Demonstrated resilience, experimentation, and technical ownership in a hardware-software project

**Tools and concepts**
Python, OpenCV, YOLOv8, computer vision, image processing, dataset labeling, real-time detection

</details>
